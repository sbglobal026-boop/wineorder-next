2. 로컬 환경변수 추가 (.env.local — git에는 안 올라감, 직접 설정 필요)
------------------------------------------------------------------

.env.local에 아래 2개 추가함:

  ADMIN_EMAILS=sbglobal026@gmail.com
  SUPABASE_SERVICE_ROLE_KEY=(Supabase 프로젝트의 service_role 키)

- ADMIN_EMAILS: admin 로그인 보호 기능(팀원이 추가)이 이 목록에 있는 이메일만
  관리자로 인정함. 없으면 로그인은 되어도 "관리자 권한이 없습니다"로 막힘.
- SUPABASE_SERVICE_ROLE_KEY: admin 상품 관리 API(/api/admin/products)가
  서버에서 service_role 키로 DB에 직접 접근하는데, 이 값이 없으면
  상품 목록이 비어있는 채로 표시됨 (API 호출이 조용히 실패).

※ 이 두 값은 Vercel 배포본에도 별도로 추가해야 함 (Project Settings →
  Environment Variables). 로컬 .env.local과 Vercel은 서로 안 연동됨.


3. 진행 중 / 아직 안 끝난 일
------------------------------

[Vercel 환경변수]
Vercel 프로젝트 → Settings → Environment Variables 에 위 2번의
ADMIN_EMAILS, SUPABASE_SERVICE_ROLE_KEY 추가 필요 (Production/Preview 체크).
추가 후 반드시 Redeploy 해야 반영됨.

[Supabase Redirect URL 설정 — 아직 미해결]
실제 배포 도메인(Vercel)에서 Google 로그인 시,
Supabase Authentication → URL Configuration의
- Site URL 이 아직 http://localhost:3000 으로 되어 있고
- Redirect URLs 에도 localhost:3000/auth/callback 만 등록되어 있어서
배포 사이트에서 로그인하면 다시 localhost:3000/?code=... 로 튕기는 문제가 있음.

해결 방법 (아직 미적용):
  1. Site URL을 실제 배포 도메인으로 변경
     예: https://your-vercel-domain.vercel.app
  2. Redirect URLs에 다음 추가:
     https://your-vercel-domain.vercel.app/auth/callback
  (localhost 항목은 로컬 개발용으로 같이 남겨둬도 됨)

→ 실제 Vercel 도메인 주소를 확인해서 Supabase에 등록해야 production
  Google 로그인이 완전히 정상화됨.
